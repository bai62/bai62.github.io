模板层

什么是模板层

模板配置

创建模板文件夹＜项目名＞/templates

在settings.py 中 TEMPLATES 配置项

  1. BACKEND：指定模板的引擎

  2. DIRS：模板的搜索目录（可以是一个或多个）

  3. APP＿DIRS：是否要在应用中的templates文件夹中搜索模板文件

  4. OPTIONS：有关模板的选项

TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates', # 模板引擎
        'DIRS': [os.path.join(BASE_DIR,'templates')], # 模板的搜索目录
        'APP_DIRS': True, # 是否要在应用中的templates文件夹中搜索模板文件
        'OPTIONS': { # 有关模板的选项
            'context_processors': [
                'django.template.context_processors.debug',
                'django.template.context_processors.request',
                'django.contrib.auth.context_processors.auth',
                'django.contrib.messages.context_processors.messages',
            ],
        },
    },
]

    os.path.join(path1[, path2[, …]]) 把目录和文件名合成一个路径

模板的加载方式

方式1

通过loader获取模板，通过HttpResponse进行响应在视图函数中

from django.http import HttpResponse

def test_template(request):
    from django.template import loader
    t = loader.get_template('test_template.html') # 返回一个template对象
    html = t.render() # 字典数据
    return HttpResponse(html)
方式2

使用render()直接加载并响应模板在视图函数中：

def test_template(request):
    from django.shortcuts import render
    return render(request,'test_template.html',) # 第三个参数是一个字典

视图层与模板层之间的交互

  1. 视图函数中可以将Python变量封装到字典中传递到模板

    样例

    def test_template(request):
    dic={
        'name':'tux',
        'test':'200',
        'end':'done',
    }
    from django.shortcuts import render
    return render(request,'test_template.html',dic) #写好字典后，不要忘记把字典传入render

  2. 模板中，我们可以用语法调用试图函数传进来的变量

    样例

    <!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
<h1>{{ name }}template test</h1>
{{ test }}{{ end }}
</body>
</html>

在模板中使用变量语法

  • { {变量名}}

  • { {变量名.index}}

  • { {变量名.key}}

  • { {对象方法}} (调用方法不需加括号)

  • { {函数名}}

示例：

#urls.py
def test_template(request):
    dic={}
    dic['int']=88
    dic['str']='string'
    dic['lst']=['sd',4,'sf',5]
    dic['dic']={'a':5,'b':'c'}
    dic['func']=say_hi
    dic['class_obj']=Dog()
    from django.shortcuts import render
    return render(request,'test_template.html',dic)

def say_hi():
    return 'hahaha'

class Dog:
    def say(self):
        return 'wangwang'
# test_template.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
<h3>int是{{ int }}</h3>
<h3>str是{{ str }}</h3>
<h3>lst是{{ lst }}</h3>
<h3>lst[0]是{{ lst.0 }}</h3>
<h3>dic是{{ dic }}</h3>
<h3>dic[a]是{{ dic.a }}</h3>
<h3>func是{{ func }}</h3>
<h3>class_obj是{{ class_obj.say }}</h3>
</body>
</html>

模板标签

作用：将一些服务器端的功能嵌入到模板中，例如流程控制等

if标签

语法：

{%if expr1%}
	...
{% elif expr2 %}
	...
{% elif expr3 %}
	...
{% else %}
	...
{% endif %}
练习:if标签
{% if x == 20 %}
x is 20
{% elif x == 30 %}
x is 30
{% endif %}

练习：if标签计算器

# mysite1/templates/mycal.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>mycal</title>
</head>
<body>
<form method="post" action="/mycal">
    <input type="text" name="x" value={{ x }}>
    <select name="op">
        <option value="add" {% if op == 'add' %}selected{% endif %}>+</option>
        <option value="sub" {% if op == 'sub' %}selected{% endif %}>-</option>
        <option value="mul" {% if op == 'mul' %}selected{% endif %}>*</option>
        <option value="div" {% if op == 'div' %}selected{% endif %}>/</option>
    </select>
    <input type="text" name="y" value="{{ y }}">={{ result }}
    <input type="submit" value="submit"> <!--submit后再次调用mycal函数-->
</form>
</body>
</html>
# mysite1/mysite1/views.py
from django.shortcuts import render

def mycal(request):
    if request.method == 'GET':
        return render(request,'mycal.html')
    elif request.method == 'POST':
        op = request.POST['op']
        x = int(request.POST['x'])
        y = int(request.POST['y'])
        result = 0

        if op == 'add':
            result = x + y
        elif op == 'sub':
            result = x - y
        elif op == 'mul':
            result = x * y
        elif op == 'div':
            result = x / y
    return render(request,'mycal.html',locals())

for标签

语法：

{% for 变量 in 可迭代对象 %}
	...循环语句
{% empty %}
	...可迭代对象无数据时填充的语句
{% endfor %}

模板过滤器

定义：在变量输出时对变量的值进行处理

作用：可以通过使用过滤器来改变变量的输出显示

语法：

{ { 变量|过滤器1:’参数值1’| 1过滤器2:’参数值2’. .}}

过滤器 说明
lower 将字符串转换为全部小写。
upper 将字符串转换为大写形式。
safe 对变量内的字符串进行html转义
add:”n” 将value的值增加n
truncatechars:’n’ 如果字符串字符多于指定的字符数量，那么会被截断。截断的字符串将以可翻译的省略号序列“…”结尾。

模板的继承

模板继承可以使父模板的内容重用，子模板直接继承父模板的全部内容并可以覆盖父模板中相应的块

语法-父模板中：

  1. 定义父模板中的块block标签

  2. 标识出哪些在子模块中是允许被修改的

  3. block标签：在父模板中定义，可以在子模板中覆盖

语法-子模版中：

继承模板 extends 标签（写在模板文件的第一行）

​ {% extends 'base.html' %}

子模板重写父模板中的内容块

{% block block_name %}
子模板块用来覆盖父模板中 block＿name 块的内容
{% endblock block_name %}

重写的覆盖规则

​ 不重写，将按照父模板的效果显示

​ 重写，则按照重写效果显示

注意

​ 模板继承时，服务器端的动态内容无法继承
